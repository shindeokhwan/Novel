---

## 문제 (Before)

매 챕터 집필 시 auto-agents 파이프라인이 **설정 파일 10~15개를 매번 전부 읽고 분석**하는 데 과도한 시간/토큰 소비:

- Lead가 Phase 1에서 reference 파일 전체를 Read
- Sub-agent(writer 3개)가 각각 `core-rules.md` + `style-guide.md` + `humanization-rules.md` 등을 다시 Read
- Sub-agent(reviewer 2개)가 `humanization-rules.md` + `plot-structure.md` + `entertainment-density.md` 등을 또 Read
- Phase 6(수정)에서 같은 레퍼런스 파일을 또다시 Read
- **결과**: 동일 파일이 파이프라인 내에서 5~8회 중복 읽기 발생

---

## 해결 전략: 3단계 최적화

### 최적화 1: core-rules.md 확장 (§9-13 추가)

**변경 전**: `core-rules.md`는 §1-8만 포함 (80줄). 인간화/쾌감/구조 규칙은 별도 파일에만 존재.

**변경 후**: 3개 reference 파일의 **핵심 규칙**을 `core-rules.md` §9-13으로 인라인:

| 추가 섹션 | 출처 | 인라인된 핵심 내용 |
|----------|------|-----------------|
| §9. 대화 진정성 | `humanization-rules.md` | 정보전달형 대사 금지, 감정 라벨링 금지, 대화 노이즈 5종, 캐릭터 음성 구분 |
| §10. 갈등 깊이+비합리성 | `humanization-rules.md` | 갈등 지속 최소, 잔여 긴장, 비합리성 쿼터 |
| §11. 에피소드 종결 5종 | `humanization-rules.md` | 끊김/대화잔향/감각단독/질문/타인시선 순환 |
| §12. 쾌감 패턴 8종 | `entertainment-density.md` | E1-E8 정의, 빈도, 연속 금지, 안티패턴, 불확실성 보존 |

**효과**: `core-rules.md`가 `.claude/rules/`에 위치하여 **모든 에이전트에 자동주입**됨. Sub-agent가 별도로 reference 파일을 Read할 필요가 사라짐.

### 최적화 2: 3-Tier 컨텍스트 로딩

**변경 전**: Phase 1에서 모든 설정 파일을 플랫하게 전부 로드 (순서 무관, 전량).

**변경 후**: 3단계 우선순위 로딩 + 조건부 로드 분리:

```
Tier 1: 항상 로드 (~30KB)
  - 현재작업상태.md
  - 직전 에피소드 마지막 1,000자만 (전체 아님)
  - ability-upgrade-current.md 마지막 20줄만 (전체 아님)

Tier 2: 등장 캐릭터만 (~20-40KB)
  - 해당 챕터 등장 캐릭터 파일만 로드
  - 비등장 캐릭터 로드 금지 ← 이전에는 전원 로드

Tier 3: 플롯 (~15KB)
  - 메인플롯 해당 챕터 섹션만 (전체 아님)
  - 서사줄기추적 최근 5챕터만
  - 복선추적 active 복선만

조건부: 해당 시에만
  - 전투씬 → style-guide-battle.md
  - 쾌감 규칙 전체 필요 → entertainment-density.md
  - 행위소/복선/바둑매핑 → plot-structure.md
```

**효과**: 기존 대비 **Read 대상 파일 40-60% 감소**. 특히 Tier 2에서 비등장 캐릭터 로드 차단이 큼 (13인 캐릭터 중 평균 4-5인만 등장).

## 핵심 원리

`core-rules.md` 자동주입 메커니즘을 활용하여, 3개 reference 파일의 핵심 규칙을 **"읽을 필요 없이 이미 알고 있는 상태"**로 만든 것이 핵심이다.

`.claude/rules/` 디렉토리의 파일은 모든 에이전트에 자동 주입되므로, 이 위치에 핵심 규칙을 인라인하면 Sub-agent가 별도 Read 없이도 규칙을 적용할 수 있다. 원본 reference 파일은 삭제하지 않고 유지하여, 금지패턴 전체 목록이나 Part별 상세 전략 등 **전량이 필요한 경우에만** 폴백으로 Read한다.
