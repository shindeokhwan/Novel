# Parallel Refactor - 5챕터 병렬 리팩토링 스킬

Sub-agent 5개로 챕터 5회분을 동시 v3 집필한 뒤, 교차 퇴고로 연관관계를 검증·수정한다.

## Trigger
- "병렬 리팩토링", "5회 병렬", "parallel refactor", "병렬 집필"

## 사용법
```
/parallel-refactor [시작챕터]-[끝챕터]
예: /parallel-refactor 3-7
```

---

## Phase 0: Pre-Flight (리드 — 병렬 투입 전 준비)

### 0-A. 범위 확인
1. 시작/끝 챕터 번호 확인 (5개 단위)
2. `03-플롯/리팩토링맵.md` 해당 챕터 블록 로드 → 챕터별 변경 유형(구조적/캐릭터/스타일) 확인
3. `03-플롯/메인플롯.md` 해당 구간 로드 → 핵심 사건, 리팩토링 변경 사항

### 0-B. 공유 컨텍스트 수집 (5개 에이전트 공통)
모든 에이전트에게 전달할 **공유 브리핑**을 구성한다.

| 항목 | 소스 | 비고 |
|------|------|------|
| 규칙 | `core-rules.md` | 자동주입됨. 별도 포함 불필요 |
| 리팩토링 룰 | Phase 0에서 수집한 해당 블록 리팩토링맵 | 에이전트에 직접 전달 |
| 직전 챕터 끝 | 시작챕터-1의 v3 마지막 1,000자 | 이음새용. 첫 에이전트에만 |
| 캐릭터 상태 | 해당 구간 등장 캐릭터 파일 요약 (핵심만 5줄 이내/인물) | 전원 공유 |
| 감정 단계 | `character-management.md` 주인공 감정 성장 아크의 해당 Part 단계 | 전원 공유 |
| 엔딩 패턴 | 직전 2챕터 엔딩 방식 | 전원 공유 (연속 금지 체크) |
| 쾌감 패턴 | 직전 2챕터 E패턴 | 전원 공유 (연속 금지 체크) |
| 악역 상태 | `villain-system.md` 해당 아크 진행 단계 | 해당 시에만 |

### 0-C. 개별 컨텍스트 수집 (챕터별 고유)
5개 챕터 각각에 대해:
1. 기존 v2 원고 전문 읽기 (리팩토링 원본)
2. 리팩토링맵에서 해당 챕터 변경 사항 추출
3. 챕터 성격 유형 / 씬 클라이맥스 / 엔딩 방식 사전 할당

### 0-D. 엔딩·쾌감 패턴 사전 배정
5개 챕터의 엔딩 방식과 E패턴을 **겹치지 않도록** 사전 배정.

| 슬롯 | 엔딩 방식 할당 | E패턴 할당 |
|------|---------------|-----------|
| Ch.N | 직전 챕터와 다른 것 | 직전 챕터와 다른 것 |
| Ch.N+1 | Ch.N과 다른 것 | Ch.N과 다른 것 |
| Ch.N+2 | Ch.N+1과 다른 것 | Ch.N+1과 다른 것 |
| Ch.N+3 | Ch.N+2와 다른 것 | Ch.N+2와 다른 것 |
| Ch.N+4 | Ch.N+3과 다른 것 | Ch.N+3과 다른 것 |

5종 엔딩(끊김/대화잔향/감각단독/질문/타인시선) 순환 배치.

## Phase 1: Parallel Writing (Sub-agent ×5 병렬)

5개 Sub-agent를 **동시 실행**한다. 각 에이전트는 1개 챕터의 v3를 독립 집필.

### 에이전트 구성
```
refactor-ch-N   : Ch.N v3 리팩토링
refactor-ch-N+1 : Ch.N+1 v3 리팩토링
refactor-ch-N+2 : Ch.N+2 v3 리팩토링
refactor-ch-N+3 : Ch.N+3 v3 리팩토링
refactor-ch-N+4 : Ch.N+4 v3 리팩토링
```

### 에이전트 브리핑 (각 에이전트에 전달)

```
## 임무
Ch.[NNN] v3 리팩토링 집필. 기존 v2를 기반으로 아래 변경 사항을 반영하여 새 버전을 작성한다.

## 공유 규칙 (core-rules.md 자동주입됨)

- "말했다" 단독 금지 (수식어/행동 비트 필수)

- 대화 중 서술 삽입 최대 2문장 (§15)
```

## Phase 2: Mechanical Scan (리드 — 기계적 검사)

5개 v3 파일에 대해 일괄 Grep 스캔. 발견 시 **즉시 수정**.

| 스캔 | Grep 패턴 | 대응 |
|------|----------|------|
| 시대착오어 11종 | `타이밍\|리듬\|패턴\|시스템\|스타일\|카리스마\|퍼센트\|이미지\|레벨\|에너지\|포인트` | 즉시 대체 |
| 수치 시간 | `0\.\d+초\|\d+초간` | 체감 표현 교체 |
| em-dash 과다 | `—` (5회 초과 카운트) | 초과분 교체 |
| 분리호칭 | `"\S+\."\s*.+(이\|가)\s*말했다` | 합치기/행동 비트 |
| "말했다" 단독 | 각 파일 "말했다" 카운트 → 3회 초과 경고 | 동사 다양화 |

## Phase 3: Cross-Chapter 퇴고 (리드 — 핵심)

5개 챕터를 **연속으로 읽으며** 연관관계를 검증·수정한다. 이 단계가 병렬 집필의 품질을 결정한다.

### 3-A. 이음새 검사 (Seam Check)
5개 v3를 순서대로 읽으며 챕터 간 이음새를 검사:
- Ch.N 마지막 200자 → Ch.N+1 첫 200자 연결 확인
- 시간/장소/상황 연속성
- POV 캐릭터의 감정 상태 연속성
- 대화 중이었다면 대화 연속성
- **불일치 발견 시 즉시 Edit**

### 3-B. 캐릭터 연속성 (Character Continuity)
가상 캐릭터(이만/소영/곽철)와 주요 조연의 5챕터 내 행동 일관성:
- 위치: 챕터 시작 시 인물이 어디에 있는지 일관적인가
- 부상 상태: 부상자가 있으면 악화/회복이 일관적인가
- 감정 궤적: 이전 챕터의 감정이 다음 챕터에서 자연스럽게 이어지는가
- **시그니처 행동**: 이만의 "먹어.", 곽철의 건방진 말투 등이 일관적인가
- 등장 빈도: 5챕터 블록 내 장기 부재 캐릭터 점검

### 3-C. 주인공 감정 그래디언트 (Protagonist Gradient)
5챕터에 걸친 주인공 감정 변화가 **점진적**인지 확인:
- 갑작스러운 성격 전환 금지 (Ch.N에서 어리바리→Ch.N+1에서 갑자기 리더)
- 성장 마커가 자연스럽게 배치되었는지
- Part별 감정 단계(character-management.md)에 부합하는지

### 3-D. 엔딩·쾌감 패턴 검증 (Pattern Verification)
Phase 0-D에서 배정한 대로 실행되었는지 확인:
- 5챕터 내 동일 엔딩 방식 연속 없는지
- 5챕터 내 동일 E패턴 연속 없는지
- 감각단독형·질문형 3연속 없는지
- **위반 발견 시 해당 챕터 엔딩 재작성 (Edit)**

### 3-E. 복선·악역 추적 (Thread Tracking)
- 악역 아크가 5챕터 블록 내에서 예정대로 진행되는지
- 복선 설치/힌트가 리팩토링맵대로 배치되었는지
- 타 진영 POV 배치가 예정대로인지

### 3-F. 톤·스타일 일관성 (Tone Consistency)
- 5챕터 전체에서 **서술 톤이 갑자기 바뀌는 구간** 없는지
- 대화 흐름 규칙(§15) 위반 구간 없는지
- 전투 차별화: 같은 전투 패턴이 반복되지 않는지
- 에이전트 간 문체 차이가 두드러지는 구간 → 리터칭

### 3-G. 수정 실행
위 3-A ~ 3-F에서 발견된 모든 문제를 **Edit 도구로 즉시 수정**.
수정량이 많은 챕터는 개별 Sub-agent로 리터칭 위임 가능.

## Phase 4: Final Assembly (리드 — 마무리)

### 4-A. 분량 검증
5개 v3 파일의 글자 수 확인:
```bash
for f in ch-NNN_v3*.md; do wc -m "$f"; done
```
- 7,000자 미만 → 확장 필수 (씬 추가 또는 내면 확장)
- 12,000자 초과 → 허용 (클라이맥스 챕터)

### 4-B. 인간화 점수 샘플링
5개 중 **2개를 무작위 선택**하여 `humanization-review` 스킬의 Phase 2-4 간이 실행.
- 70점 미만 → 해당 챕터 재작성
- 70-85점 → P1 수정만 적용
- 85점 이상 → 통과

### 4-C. 트래킹 파일 갱신
- `03-플롯/리팩토링맵.md` — 완료된 5챕터 체크
- `00-핵심관리/현재작업상태.md` — 진행 상황 업데이트
- 쾌감카운터·서사줄기추적 등은 **전체 리팩토링 완료 후 Phase 4에서 일괄 갱신**

### 4-D. 결과 리포트
```
# 병렬 리팩토링 결과 — Ch.N ~ Ch.N+4
- 완료: 5/5
- 기계적 스캔: 시대착오어 X건, em-dash X건, 말했다 X건 → 수정 완료
- 교차 퇴고: 이음새 X건, 캐릭터 연속성 X건, 패턴 위반 X건 → 수정 완료
- 인간화 샘플: Ch.X = Y점, Ch.X = Y점
- 분량: 최소 X자 ~ 최대 X자
- 다음 블록: Ch.N+5 ~ Ch.N+9
```

## 워크플로우 요약

```
Phase 0: Pre-Flight (공유/개별 컨텍스트, 패턴 사전 배정)
    ↓
Phase 1: Parallel Writing (Sub-agent ×5 동시)
    ↓ [모든 에이전트 완료 대기]
Phase 2: Mechanical Scan (Grep → 즉시 수정)
    ↓
Phase 3: Cross-Chapter 퇴고 ← 핵심 단계
    ├── 3-A: 이음새
    ├── 3-B: 캐릭터 연속성
    ├── 3-C: 감정 그래디언트
    ├── 3-D: 엔딩·쾌감 패턴
    ├── 3-E: 복선·악역
    ├── 3-F: 톤·스타일
    └── 3-G: 수정 실행
    ↓
Phase 4: Final Assembly (분량, 인간화 샘플, 트래킹)
```

## 예상 소요
- Phase 0: 리드 5분
- Phase 1: 에이전트 5개 병렬 ~10분 (병목: 가장 느린 에이전트)
- Phase 2: 리드 3분
- Phase 3: 리드 10-15분 (5챕터 순차 읽기+수정)
- Phase 4: 리드 5분
- **총: ~30-40분 / 5챕터**

## 제약
- 5챕터 블록은 **같은 소아크 내** 권장. 소아크 경계를 넘으면 톤 불일치 위험 증가.
- 구조적 변경 챕터(Tier A)는 블록 맨 앞에 배치 권장 (후속 챕터가 참조하므로).
- 가상 캐릭터 첫 등장 챕터는 해당 블록의 첫 번째로. 등장 이전 챕터와 묶지 않는다.
