# Rule Check Skill - core-rules 기반 챕터 퇴고

## 개요
지정된 챕터 파일을 core-rules.md의 모든 규칙에 대해 검사하고, 위반 부분을 수정한 뒤, 변경 내역을 사용자에게 보고한다.

## 트리거
- "룰체크", "규칙체크", "규칙 수정", "core-rules 수정", "룰 기반 퇴고"
- "ch-XXX core-rules에 따라 수정"

## 입력
- 대상 챕터 파일 경로 (예: `ch-022_v3_리팩토링.md`)
- 특정 규칙만 지정 가능 (예: "§4만 체크") — 미지정 시 전체 규칙 검사

## 워크플로우

### Step 1: 규칙 로딩
- `.claude/rules/core-rules.md` 읽기
- 체크 대상 규칙 목록 확인

### Step 2: 챕터 전문 읽기
- 대상 챕터 파일 전체 읽기

### Step 3: 규칙별 위반 검출 및 수정
아래 순서로 검사하며, 위반 발견 시 즉시 수정:

#### 3-1. §1 시대착오어 (11종)
- 타이밍, 리듬, 패턴, 시스템, 스타일, 카리스마, 퍼센트, 이미지, 레벨, 에너지, 포인트
- 추가: 현대 경영/IT/스포츠 용어 (기업, 합병, 카운터, 스윙, 톤 등)
- 수정 시 시대에 맞는 한자어 또는 고유어로 대체

#### 3-2. §4 문장 단편화
- SVO 단문 3문장 이상 연속 → 접속어미로 통합
- 정보 쪼개기: 한 문장으로 합칠 수 있는 동작+대상+속성 분절 → 합침
- 주어 중복 재소개 → 합치거나 주어 생략
- 어미 단조 반복(~했다/~이었다) 3문장 이상 연속 → 어미 변형
- 시나리오식 컷 나열(한 줄 독립 단락 연속) → 단락 내 흐름으로 통합

#### 3-3. §5 대화 패턴
- 분리호칭 ("이름." 누가 말했다. "내용") → 합치기 또는 행동 비트
- "[대사]." [이름]이 말했다. 단독 → 수식어/행동 비트 추가

#### 3-4. §7 기본 서술 원칙
- Show Don't Tell: 감정 직접 명명 → 신체 반응/행동으로
- 수치 시간("0.1초") → 체감적 표현

#### 3-5. §9 대화 진정성
- 정보전달형 대사 / 감정 라벨링 대사 / 서술형 대사 금지
- 리액션 중복 금지 (한 비트에 동작 1개만)
- 에코 대사 금지
- 캐릭터 음량 일관 (과묵한 캐릭터의 장문 등)

#### 3-6. §19 선행맥락 규칙
- 대사/행동/사건참조/비교 → 선행 서술 확인

#### 3-7. §20 캐릭터 정보 경계
- 캐릭터가 모르는 정보를 아는 것처럼 말하는 경우
- "연의" 등 비밀 정보가 타인 앞에서 언급되는 경우

#### 3-8. §21 존비어 일관성
- 한 캐릭터 대사 내 존대/반말 혼용
- 호칭 전환 시점 불일치 (두목/현령 이후 장삼 사용 등)

#### 3-9. §22 능력 성장 점진성
- 능력 급성장 (한계→수련/계기→확장 3단계 누락)

#### 3-10. §23 전투 종결의 물리적 인과
- 부상 연속성 / 종결 무게 비례 / 물리적 근거

### Step 4: 보고
수정 완료 후, 사용자에게 아래 형식으로 보고:

```
## 수정 보고서 — [파일명]

### §X [규칙명] (N건)
- 행 XX: "원문" → "수정문" — [위반 사유]

### 위반 없음 확인
- §X, §X: 위반 없음
```

## 주의사항
- 수정문을 작성할 때도 core-rules를 준수할 것 (시대착오어, 존비어 등)
- 의미를 바꾸지 말고 문장 구조/단어만 개선
- 이전 챕터와의 설정 일관성도 확인 (부상 상태, 호칭, 기 단계 등)
- 수정 내역은 반드시 사용자에게 명시적으로 보고
