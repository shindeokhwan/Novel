# Plot Structure Rules - 플롯/서사 구조 규칙

서사 모델, 행위소 모델, 복선 관리, 바둑-전략 매핑을 정의한다.

## Primary Narrative Model: MICE-Character

성장물의 핵심 서사 모델. 불만족한 캐릭터 → 변화 시도 → 새 자아/위치.

### MICE 쿼텟 적용
- **Character (1차)**: 졸병 → 통일자 성장 아크
- **Event (2차)**: 황건적의 난 → 삼국 정립 → 통일 (세계 균형 복원)
- **Idea (3차)**: 역사를 바꿀 수 있는가? 이 삶은 나의 것인가?
- **Milieu (4차)**: 현대 → 삼국시대 (새 세계 적응). 회귀 원인은 미스터리(의도적 미해결)

### 적용 원칙
- 먼저 열린 쓰레드가 나중에 닫힌다 (LIFO 구조)
- 성장 아크가 모든 쓰레드를 관통
- 동시 활성 쓰레드는 최대 3-4개

## Secondary Models

### Braid Structure (엮음 구조)
- 다중 세력의 이야기가 교차하며 엮임
- 주인공 줄기 + 적 진영 줄기 + 제3세력 줄기
- 교차점에서 충돌/동맹 발생

### Rise Structure (성장물 구조)
- 단계적 성장의 리듬감
- 각 단계마다: 도전 → 위기 → 극복 → 보상 → 새 도전
- 성장 단계: 졸병 → 두목 → 현령 → 태수 → 주목 → 왕 → 통일

## 바둑-서사 대응 구조

| 바둑 단계 | Part | 서사 단계 | 핵심 행동 |
|----------|------|----------|----------|
| 포석(布石) | Part 1-2 | 세력 구축 | 거점 확보, 인재 영입, 동맹 |
| 중반전(中盤) | Part 3 | 세력 충돌 | 대규모 전쟁, 영토 쟁탈 |
| 끝내기(ヨセ) | Part 4 | 통일 마무리 | 최종 영토 확정, 평정 |

### 바둑 전략 → 군사 전략 매핑
- **포석 실패** = 불리한 전쟁 (더 큰 역전이 필요)
- **수읽기 적중** = 적 전략 예측 성공 → 매복/기습
- **대국관** = 전체 전장 조감 → 핵심 거점 식별
- **사활 판단** = 포위당한 아군 구출 여부 결정
- **패싸움** = 교착 상태 → 희생과 맞교환
- **기풍(棋風)** = 전투 전략 원형 — 이세돌(침투·돌파), 이창호(견고·소모), 조훈현(세력·전격), 유창혁/서봉수(난전·백병). 규모별 적용: 소규모=수읽기, 중규모=기풍 선언, 대규모=기풍 충돌+부대 포석. `style-guide.md` 바둑 기풍 전투 시스템 섹션 참조.

## P1. 챕터 성격 선언 시스템 (반복성 해소)

집필 전 5가지 유형 중 **하나를 명시적으로 선택**. `writing-process.md` Before Writing에 선언.

| 유형 | 정의 | 씬 구성 기본형 |
|------|------|--------------|
| **포석형** | 정보 수집, 계획, 관계 형성 | 설정→심화→불확실성 잔존 |
| **돌파형** | 직접 행동, 전투, 결단 | 긴장→폭발→여진 |
| **여운형** | 직전 사건 처리, 감정 정리 | 상태 확인→해석→새 의문 발아 |
| **인간형** | 전략 없이 사람과 사람의 이야기 | 일상→균열→온기 |
| **전환형** | 기존 계획을 뒤집는 사건 | 예상→파열→재계산 |

### 연속 규칙
- 같은 유형 **3회 연속 금지** (현재 포석형 과다 사용 중)
- **인간형**: 5챕터당 1회 이상 필수 (전략적 진전 없이 인간적 장면만)
- **전환형**: 소아크당 2회 이상 필수

## P2. 씬 클라이맥스 위치 선택제 (반복성 해소)

집필 전 3가지 패턴 중 하나를 명시적으로 선택.

| 배치 | 구조 | 독자 경험 |
|------|------|---------|
| **전면배치** | 씬1=사건 → 씬2=파장 → 씬3=새 질문 | "시작부터 뭔가 터졌다, 여운이 남는다" |
| **중간배치** | 씬1=축적 → 씬2=폭발 → 씬3=여운/처리 | "중간에 터지고 끝까지 울린다" |
| **후면배치** | 씬1=준비 → 씬2=심화 → 씬3=카타르시스 | 현재 기본값 |

### 배치 규칙
- **후면배치만 연속 4회 금지**
- 5챕터당: 전면배치 1회, 중간배치 1회 이상 필수

## P7. 캐릭터 관계 클럭 시스템 (반복성 해소)

합류 후 "역할 고정" 문제 해소. 각 인물의 관계는 2~3회 진동 후 안정화되어야 진짜 충성심이 의미를 갖는다.

### 클럭 주기
- 합류 후 5~8챕터 내: **첫 번째 균열** (배신 아님 — 가치관 차이, 방법론 충돌)
- 균열 후 3~5챕터 내: **심화 사건** (같은 위기를 함께 겪음)
- 2~3회 진동 후: **안정화** (이 안정화가 진짜 충성심)

### Ch.55-65 적용 대상
| 인물 | 합류 챕터 | 균열 상태 | 배치 필요 시점 |
|------|---------|---------|-------------|
| 태사자 | Ch.43 | 균열 없음 | Ch.57~59 |
| 주방 | Ch.37 | 균열 없음 | Ch.55~57 |
| 전예 | Ch.29 | 균열 없음 | Ch.58~60 |

## P8. 소아크별 핵심 감정 / 장르 정체성 (반복성 해소)

각 소아크가 독자에게 전달하는 **지배적 감정과 장르적 질감**을 사전에 명시. 소아크 전환점이 진짜로 느껴지려면 질감 자체가 달라야 한다.

| 소아크 | 지배 감정 | 장르적 질감 | 상태 |
|--------|---------|-----------|------|
| 2-1 포석기 | 조심스러운 야망 → 첫 시험 통과 | 성장물 | 완료 ✓ |
| 2-2 확장기 (Ch.36-65) | **야망 과잉 → 부상 → 역설적 성숙** | 역경 극복물 | 진행 중 |
| 3-1 관도 | 거인들 틈의 불안 → 내부 배신 공포 | **스릴러** | 미시작 |
| 3-2 팽창기 | 대결의 쾌감 → 한계 직면 | 서사 액션 | 미시작 |
| 4-1 끝내기 1기 | 통일의 무게감 + 이별 예감 | **비극적 서사** | 미시작 |

### 소아크 2-2 남은 챕터(Ch.55-65) 기저 감정
**"부상이라는 제약 속에서 더 넓은 세계를 쥐는 역설"** — 이것이 모든 씬의 기저 감정이어야 함.
발기 달성(Ch.58)이 단순 능력 회복이 아니라 "제약을 통해 얻은 깊이로 더 높이 오르는" 역설로 독자에게 전달되어야 한다.

## Three-Act Structure per Part

```
각 Part 내부:
1막 (설정): ~25% | 2막 (대립): ~50% | 3막 (해결): ~25%
```

## Actantial Model (행위소 모델)

### 전체 이야기
- **주체**: 서진우/장삼 (주인공)
- **대상**: 삼국 통일 + 태평성세
- **발신자**: 회귀 (운명/AI 폭주)
- **수신자**: 삼국시대 백성들
- **조력자**: 영입한 인재들, 바둑적 사고
- **대립자**: 조조/유비/손권 세력, 역사의 관성

### 추적 규칙
- 전체 이야기와 현재 챕터 각각 행위소 모델 추적
- 역할 전환 (조력자→대립자 등) 발생 시 기록

## Foreshadowing Management

### 복선 생명주기
```
설치 → 발전 → 힌트 → 회수
 ↓       ↓      ↓      ↓
[은밀] [미묘] [암시] [공개]
```

### 복선 타입
- **바둑**: 바둑 용어가 나중에 전략으로 실현
- **역사**: 역사적 사건의 전조, 변경될 역사의 암시
- **정체성**: 장삼→서홍→서진우의 정체성 여정, 이 삶의 의미
- **캐릭터**: 숨겨진 정체, 배신, 숨은 능력
- **세계관**: 기(氣)의 비밀, 역사 이면의 진실

### 규칙
- 모든 복선은 `03-플롯/복선추적.md`에 즉시 등록
- 미회수 복선은 결말 전 모두 회수 또는 의도적 미해결 표시
- 바둑 복선은 반드시 전략으로 회수

## Plot Hole Prevention

### 확인 항목
- 인과관계: 모든 사건에 합리적 원인
- 동기: 캐릭터 행동에 납득할 동기
- 역사: 변경된 역사의 파급효과 추적
- 군사: 병력 숫자, 보급, 이동 시간의 현실성
- 무예: 전투 결과와 무예 수준의 정합성
- 정보: 캐릭터가 아는 정보의 출처 명확

### 발견 시 처리
1. `03-플롯/플롯홀.md`에 즉시 기록
2. 심각도 평가 (Critical/Major/Minor)
3. 수정 방안 3개 이상 제안
4. 수정 후 관련 파일 일괄 업데이트
