# Plot Structure Rules - 플롯/서사 구조 규칙

서사 모델, 행위소 모델, 복선 관리, 바둑-전략 매핑을 정의한다.

## Primary Narrative Model: MICE-Character

성장물의 핵심 서사 모델. 불만족한 캐릭터 → 변화 시도 → 새 자아/위치.

### MICE 쿼텟 적용
- **Character (1차)**: 졸병 → 통일자 성장 아크
- **Event (2차)**: 황건적의 난 → 삼국 정립 → 통일 (세계 균형 복원)
- **Idea (3차)**: 회귀의 의미, 역사를 바꿀 수 있는가?
- **Milieu (4차)**: 현대 → 삼국시대 (새 세계 적응)

### 적용 원칙
- 먼저 열린 쓰레드가 나중에 닫힌다 (LIFO 구조)
- 성장 아크가 모든 쓰레드를 관통
- 동시 활성 쓰레드는 최대 3-4개

## Secondary Models

### Braid Structure (엮음 구조)
- 다중 세력의 이야기가 교차하며 엮임
- 주인공 줄기 + 적 진영 줄기 + 제3세력 줄기
- 교차점에서 충돌/동맹 발생

### Rise Structure (성장물 구조)
- 단계적 성장의 리듬감
- 각 단계마다: 도전 → 위기 → 극복 → 보상 → 새 도전
- 성장 단계: 졸병 → 두목 → 현령 → 태수 → 주목 → 왕 → 통일

## 바둑-서사 대응 구조

| 바둑 단계 | Part | 서사 단계 | 핵심 행동 |
|----------|------|----------|----------|
| 포석(布石) | Part 1-2 | 세력 구축 | 거점 확보, 인재 영입, 동맹 |
| 중반전(中盤) | Part 3 | 세력 충돌 | 대규모 전쟁, 영토 쟁탈 |
| 끝내기(ヨセ) | Part 4 | 통일 마무리 | 최종 영토 확정, 평정 |

### 바둑 전략 → 군사 전략 매핑
- **포석 실패** = 불리한 전쟁 (더 큰 역전이 필요)
- **수읽기 적중** = 적 전략 예측 성공 → 매복/기습
- **대국관** = 전체 전장 조감 → 핵심 거점 식별
- **사활 판단** = 포위당한 아군 구출 여부 결정
- **패싸움** = 교착 상태 → 희생과 맞교환

## Three-Act Structure per Part

```
각 Part 내부:
1막 (설정): ~25% | 2막 (대립): ~50% | 3막 (해결): ~25%
```

## Actantial Model (행위소 모델)

### 전체 이야기
- **주체**: 서진우/장삼 (주인공)
- **대상**: 삼국 통일 + 태평성세
- **발신자**: 회귀 (운명/AI 폭주)
- **수신자**: 삼국시대 백성들
- **조력자**: 영입한 인재들, 바둑적 사고
- **대립자**: 조조/유비/손권 세력, 역사의 관성

### 추적 규칙
- 전체 이야기와 현재 챕터 각각 행위소 모델 추적
- 역할 전환 (조력자→대립자 등) 발생 시 기록

## Foreshadowing Management

### 복선 생명주기
```
설치 → 발전 → 힌트 → 회수
 ↓       ↓      ↓      ↓
[은밀] [미묘] [암시] [공개]
```

### 복선 타입
- **바둑**: 바둑 용어가 나중에 전략으로 실현
- **역사**: 역사적 사건의 전조, 변경될 역사의 암시
- **회귀**: AI 폭주의 진실, 돌아갈 방법, 회귀의 목적
- **캐릭터**: 숨겨진 정체, 배신, 숨은 능력
- **세계관**: 기(氣)의 비밀, 역사 이면의 진실

### 규칙
- 모든 복선은 `03-플롯/복선추적.md`에 즉시 등록
- 미회수 복선은 결말 전 모두 회수 또는 의도적 미해결 표시
- 바둑 복선은 반드시 전략으로 회수

## Plot Hole Prevention

### 확인 항목
- 인과관계: 모든 사건에 합리적 원인
- 동기: 캐릭터 행동에 납득할 동기
- 역사: 변경된 역사의 파급효과 추적
- 군사: 병력 숫자, 보급, 이동 시간의 현실성
- 무예: 전투 결과와 무예 수준의 정합성
- 정보: 캐릭터가 아는 정보의 출처 명확

### 발견 시 처리
1. `03-플롯/플롯홀.md`에 즉시 기록
2. 심각도 평가 (Critical/Major/Minor)
3. 수정 방안 3개 이상 제안
4. 수정 후 관련 파일 일괄 업데이트
